# Красно-чёрное дерево и Ассоциативный массив (словарь)

Реализация самобалансирующегося красно-чёрного дерева и ассоциативного массива (словаря) на его основе.

# Что реализовано

Красно-чёрное дерево (RBTree) - cамобалансирующееся бинарное дерево поиска, гарантирующее высоту O(log n) и, соответственно, логарифмическое время всех основных операций в худшем случае.

Операции:  
rb_insert - вставка пары (ключ, значение)  
rb_find - поиск значения по ключу  
rb_contains - проверка наличия ключа  
rb_remove - удаление по ключу  
rb_size - количество элементов  
rb_check_properties - проверка всех пяти красно-чёрных инвариантов  
rb_black_height - получение чёрной высоты дерева  

Ассоциативный массив / Словарь (Dict) - cтруктура данных, хранящая пары ключ-значение и построенная поверх красно-чёрного дерева.  
В отличие от хеш-таблицы, гарантирует упорядоченное хранение ключей и O(log n) в худшем случае.

Операции:  
dict_put - вставка или обновление пары (ключ, значение)  
dict_get - получение значения по ключу  
dict_contains - проверка наличия ключа  
dict_remove - удаление по ключу  
dict_size - количество элементов  
dict_print - печать всех пар в порядке возрастания ключей  

# Бинарное дерево поиска (BST)

Это структура, состоящая из узлов. У каждого узла есть ключ и не более двух потомков - левый и правый.  
Правило: все ключи в левом поддереве меньше ключа узла, все ключи в правом - больше.  

Позволяет находить элемент за O(log n) в сбалансированном случае.  
Однако при вставке отсортированных данных (1, 2, 3, …) дерево вырождается в связный список, и сложность скатывается до O(n).

# Красно-чёрное дерево (Red-Black Tree)

Это самобалансирующееся бинарное дерево поиска, в котором каждый узел хранит дополнительный бит — цвет (красный или чёрный).  
Дерево обязано удовлетворять пяти инвариантам:  

1. Свойство цвета: каждый узел либо красный, либо чёрный.  
2. Свойство корня: корень всегда чёрный.  
3. Свойство листьев: все листья (nil-потомки) считаются чёрными.  
4. Свойство красного узла: оба потомка красного узла обязательно чёрные. Не бывает двух красных узлов подряд.  
5. Свойство чёрной высоты: для любого узла все простые пути от него до листьев содержат одинаковое количество чёрных узлов.

Из этих правил математически выводится, что самый длинный путь от корня до листа не более чем вдвое превышает самый короткий. Это гарантирует O(log n) для высоты дерева.

Как достигается баланс:  
При вставке или удалении после обычной BST-операции проверяется, не нарушились ли инварианты, и при необходимости выполняются:  
Перекрашивание - смена цвета узла  
Повороты - левый и правый, меняющие структуру дерева без нарушения порядка ключей  

# Ассоциативный массив (словарь)  

Это абстрактный тип данных «ключ - значение».  
Интерфейс: put, get, remove, contains.  

# Структура проекта  

rb_tree.h - заголовочный файл красно-чёрного дерева (структуры, прототипы)  
rb_tree.c - реализация КЧ-дерева: вставка, удаление, поиск, балансировка, проверка свойств   
dict.h - заголовочный файл словаря   
dict.c - реализация словаря поверх КЧ-дерева   
main.c - автоматические тесты и примеры использования   
README.md - этот файл   

# Сборка и запуск  

Проект написан на чистом C (стандарт C99), не требует внешних зависимостей. Сборка - одним вызовом компилятора.  

gcc -std=c99 -Wall -Wextra -o test main.c dict.c rb_tree.c  
./test или test.exe
